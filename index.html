When to use

When NOT to use

Section 02

Motion Tokens

All animations use GSAP v3. Do not invent new durations or easings — use these tokens. Every animation must check prefers-reduced-motion.

Durations

Token Value Visual scale Use
instant 100ms / 0.1s
Active/pressed states
fast 150ms / 0.15s
Hover, toggles, tooltips
normal 300ms / 0.3s
Modals, dropdowns, fades
slow 500ms / 0.5s
Page sections, large reveals
slower 700ms / 0.7s
Staggered sequences
slowest 1000ms / 1.0s
Hero/cinematic, logo reveals

Easings

Name GSAP Use
out (default) power2.out Entrances, general UI transitions
in power2.in Exit animations
in_out power2.inOut Layout shifts, theme toggles
overshoot back.out(1.7) Modals, playful bounces
spring elastic.out(1, 0.3) Springy interactions — GSAP only
bounce bounce.out Bounce landings — GSAP only
Reduced Motion — Mandatory
Every animation must check window.matchMedia("(prefers-reduced-motion: reduce)").matches. When active: set all GSAP durations to 0, disable ScrollTrigger scrub/parallax, limit opacity fades to 150ms max, disable all transform animations.

Section 04

Component Customizations

If a component isn't listed here, use shadcn exactly as provided. Never edit files inside components/ui/ — the shadcn CLI must remain updatable.

Animation Assignments

Component Entrance Exit Notes
Dialog / AlertDialog scaleIn scaleOut Slight scale with overshoot
Sheet (left) slideInLeft slideOutLeft Direction matches open side
Sheet (right) slideInRight slideOutRight Direction matches open side
DropdownMenu slideInDown fadeOut Drops down, fades out quickly
Popover scaleIn fadeOut Scale from trigger point
Tooltip fadeIn 150ms fadeOut 150ms Fast — never distracting
Toast slideInRight slideOutUp Slides in, dismisses upward
Accordion expandIn reverse Smooth height from top
Tabs fadeIn 150ms — Quick crossfade

Style Overrides

Button
Hover: 150ms ease-out transition
Active: scale(0.97) press effect
Card
Default: shadow-sm
Interactive hover: shadow-md + lift

Uses shadcn --ring variable

Section 05

Get Started

Set up a new project with shadcn/ui and this design system in five steps.

1
Scaffold your framework
# Next.js npx create-next-app@latest my-app --typescript --tailwind --eslint
2
Install shadcn/ui (Neutral theme)
npx shadcn@latest init # Choose: Neutral base color, CSS variables: yes
3
Install GSAP
npm install gsap
4
Copy brand tokens into your global stylesheet
/* Copy tokens/colors.css — Section 2 block into globals.css */ :root { --brand-surface: #FAFAFA; --brand-highlight-yellow: #FFD400; --brand-highlight-blue: #139CFF; --brand-highlight-magenta: #FF68E3; } .dark { --brand-surface: #151515; --brand-highlight-yellow: #F9DE0A; --brand-highlight-blue: #2BC3FF; --brand-highlight-magenta: #FF68FF; }
5
Import animation presets
import { ENTRANCE, EXIT, HOVER, applyPreset } from "@/design-system/animations/presets"; // Apply a preset applyPreset(gsap, element, "slideInUp"); // Or use directly gsap.fromTo(el, ENTRANCE.scaleIn.from, ENTRANCE.scaleIn.to);
